# ContractGuard 产品闭环审计与实现验收

审计日期：2026-07-18  
实现验收：2026-07-22  
范围：已登录成员从工作台进入审阅结果，并判断下一步该做什么。  
用户目标：把 AI 审阅结果转化为可分派、可追踪、可批准的团队工作。

## 总体判断

现有界面清晰、稳定，风险证据和隐私说明也比普通演示项目完整；“像 Demo”的主要原因不是视觉粗糙，而是流程在报告生成后终止。团队看不到负责人、截止时间、处置进度、审批状态和活动历史，也无法从工作台回答“今天谁要处理什么”。

## 步骤与健康度

1. **工作台 — 基础良好，但缺少运营入口。**
   - 证据：[01-dashboard.png](01-dashboard.png)
   - 优点：层级明确，累计审阅、完成、高风险和失败状态易扫读。
   - 风险：指标全部围绕“生成了多少报告”，没有我的待办、逾期、高风险未关闭或待审批。
2. **审阅详情 — 证据充分，但没有业务闭环。**
   - 证据：[02-review-detail.png](02-review-detail.png)
   - 优点：风险、原文证据、建议和义务分区清楚，风险等级可筛选。
   - 风险：每条风险只有复制建议和模型质量反馈；义务只是静态清单。没有指派、截止日期、状态、处置理由或批准门禁。
3. **新建审阅 — 输入路径清楚，交接信息不足。**
   - 证据：[03-new-review.png](03-new-review.png)
   - 优点：粘贴/上传模式清楚，保存说明和人工复核提醒建立了信任。
   - 风险：提交后直接进入结果，没有形成后续团队工作；同步等待也不适合长文档和高并发场景。

## 最高优先级改进

本轮应先把产品从“报告查看器”升级为“合同风险运营工作台”：

- 风险与义务自动物化为统一工作项。
- 工作项支持负责人、明确截止时间、状态、备注和事件时间线。
- 审阅支持提交审批、批准、退回；高风险未解决或未接受时禁止批准。
- 工作台增加我的未完成、已逾期、高风险未关闭、待审批和本周完成。

修订版本差异、异步任务队列和通知提醒适合后续阶段，不应与本轮核心闭环同时摊开。

## 可访问性观察与证据边界

- 截图与 DOM 显示已有跳转到主要内容、语义区域、标题层级、按钮名称和筛选选中态，这是良好基础。
- 长详情页需要确保新增状态不仅依赖颜色，弹窗具备标题、标签、关闭路径和键盘焦点管理。
- 截图无法证明完整键盘顺序、焦点可见性、读屏播报、对比度数值或缩放到 200% 后的全部行为；这些必须在实现后的浏览器验收中单独验证。

## 实现后的闭环

本轮已把上述最高优先级改进全部落地：完成审阅会幂等生成风险与义务工作项，统一进入处置中心；工作项可指派负责人、设置明确截止时间、填写处置说明、按状态流转并查看事件时间线；审阅可提交、驳回或批准，未解决/未接受的高风险会阻止批准。所有可变状态都使用乐观锁，服务端权限和前端可见状态保持一致。

- 审阅详情与审批闭环：[04-review-operations.png](04-review-operations.png)
- 处置中心与组合筛选：[05-operations-center.png](05-operations-center.png)

与 [02-review-detail.png](02-review-detail.png) 对比，新的审批区在摘要之前明确呈现业务状态；每条风险和义务卡片都显示处置状态、负责人、截止时间及动作入口。风险证据、建议和原有视觉语言保持不变，没有把工作流割裂为另一套界面。

## 浏览器验收步骤与结果

1. **owner 注册并从示例合同创建真实审阅 — 健康。** 审阅完成后生成 4 条风险工作项和 2 条义务工作项，详情页正确显示 `0/4` 与 `0/2` 闭环进度。
2. **必填说明校验 — 健康。** 风险直接转为“已解决”但未填写说明时，前端阻止保存并把焦点留在说明字段。
3. **负责人、截止时间、状态与事件 — 健康。** 指派 owner、设置明确时间、填写说明并保存后，卡片和处置中心立即更新；时间线同时保留物化事件和状态变更事件。
4. **高风险审批门禁 — 健康。** 草稿可提交为审批中；仍有 1 条高风险未关闭时，批准返回中文阻塞说明。最后一条高风险被接受后，旧错误会自动清除，随后批准成功。
5. **处置中心筛选 — 健康。** 首次加载显示全部 6 项；组合“风险处置 + 待处理 + 尚未指派”后精确显示 2 项。空筛选值不会再作为非法枚举发送。
6. **只读角色 — 健康。** viewer 看不到新建审阅和成员管理入口，队列只显示“查看工作项”；弹窗字段均为 disabled、没有保存按钮，审批区只有只读说明。
7. **并发与服务端权限 — 健康。** API 自动化验证旧 `expected_version` 返回 `409`，reviewer 不能重开已批准审阅，viewer 写入返回 `403`，owner/admin 才能接受风险、豁免义务或做最终决定。
8. **运行与视觉检查 — 健康。** 浏览器控制台没有应用错误；桌面视口没有横向溢出。新增页面复用现有颜色、间距、圆角、字体和焦点样式，并提供 1080、700、480 三档响应式规则。

## 验收边界

本轮在应用内浏览器的固定桌面视口完成交互和视觉验收；移动端通过响应式规则、DOM 语义和最小宽度约束做了代码级复核，但浏览器控制面未提供视口缩放接口，因此 375px 真机截图仍应作为发布前设备矩阵的一部分。长文档仍是同步请求，异步队列、通知、版本对比和 SLA 监控属于下一阶段能力。
